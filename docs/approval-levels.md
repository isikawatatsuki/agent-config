# 承認レベル定義

エージェントが実行するアクションに対する承認レベルを定義する。

## 承認レベル

| レベル | 名称 | 説明 |
|--------|------|------|
| **L0** | 自動実行 | 承認不要。エージェントが自律的に実行する |
| **L1** | 通知のみ | 実行後にDiscordルームへ結果を報告する |
| **L2** | 事前承認 | 実行前にDiscordルームへ内容を送り、承認を得てから実行する |
| **L3** | 禁止 | エージェントは実行しない。人間が手動で行う |

## アクション別承認レベル

### コード・ファイル操作

| アクション | レベル | 備考 |
|------------|--------|------|
| ファイル読み取り・検索 | L0 | 常に自動実行 |
| ファイル編集・作成 | L0 | ブランチ上での作業に限る |
| ファイル削除 | L1 | 削除内容を報告 |

### Git 操作

| アクション | レベル | 備考 |
|------------|--------|------|
| ブランチ作成 | L0 | 自動実行 |
| コミット | L0 | ブランチ上で自動実行 |
| PR 作成 | L1 | 作成後にリンクを報告 |
| PR マージ | L2 | マージ前に承認を得る |
| ブランチ削除 | L1 | マージ済みブランチのみ。報告する |
| force push | L3 | 禁止。人間が判断する |
| main への直接プッシュ | L3 | 禁止。必ずPR経由 |

### GitHub 操作

| アクション | レベル | 備考 |
|------------|--------|------|
| Issue 作成 | L0 | 自動実行 |
| Issue 更新・クローズ | L1 | 変更内容を報告 |
| Issue 削除 | L3 | 禁止 |
| PR レビューコメント | L0 | 自動実行 |

### 外部サービス・通知

| アクション | レベル | 備考 |
|------------|--------|------|
| Discord ルームへの通知 | L0 | 通常の報告・応答 |
| Web ページの取得 | L0 | 自動実行 |
| 外部 API の呼び出し（読み取り） | L0 | GET 系のみ |
| 外部 API の呼び出し（書き込み） | L2 | 事前承認が必要 |

### インフラ・デプロイ

| アクション | レベル | 備考 |
|------------|--------|------|
| デプロイ実行 | L2 | 事前承認が必要 |
| インフラ変更（IaC） | L2 | 事前承認が必要 |
| データベース操作 | L3 | 禁止。人間が実行する |
| 環境変数・シークレットの変更 | L3 | 禁止。人間が実行する |

### 破壊的操作

| アクション | レベル | 備考 |
|------------|--------|------|
| `rm -rf` 等の大量削除 | L3 | 禁止 |
| データの不可逆な変更 | L3 | 禁止 |
| 権限・アクセス制御の変更 | L3 | 禁止 |

## 承認リクエストのフォーマット

L2（事前承認）のアクションを実行する場合、以下の形式でDiscordルームに送る：

```
🔐 承認リクエスト

アクション: <実行するアクション>
対象: <対象リソース>

内容:
```
<実行内容の詳細>
```

理由: <なぜこのアクションが必要か>
リスク: <想定されるリスク>
```

## 承認レベルの変更

- 承認レベルの変更はこのファイルを更新するPRを作成し、承認を得てから反映する
- 一時的なレベル変更（例: 特定タスクでの L2 → L0 昇格）は、Discordルームでの口頭承認で可とする
- 一時的な変更は該当タスク完了後に自動的に元のレベルに戻る

## Loop Engineering との対応

| Loop Engineering | 承認レベル |
|------------------|-----------|
| L1 (Report-only) | エージェントの全アクションは L0〜L1 の範囲 |
| L2 (Assisted) | 修正提案は L1、マージ等は L2 |
| L3 (Unattended) | 低リスクアクションを L0 に昇格 |
